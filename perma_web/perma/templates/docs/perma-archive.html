Archive Formats

Perma provides multiple archive formats for each archived web page. You can navigate through the different formats using the tabs at the top of each page. 


Live Page View
The cited web page as it currently exists.

In the Live Page View, you’ll see the live, present-day contents of the original source URL. This view, which is not an archive, allows you to check quickly to see if the original page is still live and, if so, you can navigate directly to the live site.

If the Live Site View reveals a 404 error or the original web page has changed, you can use the tabs to select among available archive formats for the cited web page.


Screen Cap·ture View
An image of the cited webpage as it existed when the Perma Link was created.

If you select the Screen Capture tab, you’ll see a screenshot of the original web page as it appeared when the Perma Link was created. If the Perma Link is unvested, you will see a blue Perma icon overlay on the image.


Ar·chived Page View
The webpage, rendered in html & css, with accompanying images, that best recreates the view and functionality as it existed when the Perma Link was created.

If you select the Archived Page tab, you’ll see an interactive archive of the cited web page as it appeared when the Perma Link was created. If the Perma Link is unvested, you will see the blue Perma icon overlay on the archived pages.


Public Archive

Pub·lic Ar·chive
A repository for publicly viewable materials archived using Perma.

Most of the materials archived by Perma users are part of the Public Archive, which means the materials are publicly accessible to anyone who clicks the associated Perma Link or enters the URL for that Perma Link into their browser.


Dark Archive

Dark Ar·chive
A repository for materials archived using Perma, but which are not publicly viewable.

Some of the materials archived by Perma users become part of the Dark Archive, which means they are preserved but not widely accessible. These materials remain available to the person who created the Perma Link. Once vested, these materials also are available to the Vesting User, Vesting Organization and Registrar responsible for the vesting decision.

Materials may become part of the Dark Archive for different reasons. The person who created or vested a Perma Link may designate it for the Dark Archive. Also, by default, Perma places in the Dark Archive any pages with a “noarchive” metatag or a Perma-specific exclusion in the robots.txt file. Finally, archived materials are placed in the Dark Archive upon receipt of a valid DMCA takedown notice.
